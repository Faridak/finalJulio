Authentication

The VentDepot API supports both JWT Bearer tokens and API keys for authentication.

POST /auth/login

Authenticate user and receive JWT token

Request Body:

{
  "email": "user@example.com",
  "password": "userpassword"
}

Response:

{
  "success": true,
  "user": {
    "id": 1,
    "email": "user@example.com",
    "role": "customer"
  },
  "token": "eyJ0eXAiOiJKV1QiLCJhbGc...",
  "expires_in": 3600
}

Using Authentication

Include the token in the Authorization header:

Authorization: Bearer YOUR_JWT_TOKEN

Or use an API key:

X-API-Key: YOUR_API_KEY

Products

GET /products

Retrieve products with filtering and pagination

Query Parameters:

Parameter Type Description
page integer Page number (default: 1)
limit integer Items per page (max: 50, default: 20)
category string Filter by category
search string Search in name and description
merchant_id integer Filter by merchant
sort string Sort field (name, price, created_at, average_rating)
order string Sort order (asc, desc)

Example Response:

{
  "success": true,
  "data": [
    {
      "id": 1,
      "name": "Product Name",
      "description": "Product description",
      "price": 29.99,
      "category": "Electronics",
      "merchant_id": 2,
      "merchant_email": "merchant@example.com",
      "merchant_name": "John Doe",
      "average_rating": 4.5,
      "review_count": 25,
      "created_at": "2024-01-01 12:00:00"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 100,
    "pages": 5
  }
}

Error Codes

Code Status Description
200 OK Request successful
400 Bad Request Invalid request data
401 Unauthorized Authentication required
403 Forbidden Insufficient permissions
404 Not Found Resource not found
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Server error

Rate Limits

Current Limits

  • 100 requests per minute per IP address
  • 5 login attempts per 5 minutes per IP
  • Higher limits available for authenticated users
